# KD Дерево
## Теория
### Определения
- KD Дерево - особая структура данных специально создания для хранения точек в $k$ мерном пространстве. Данное Дерево является логическим потомком _бинарного сбалансированного дерева_ в котором, каждый "слой" делит пространство с точками на две части по разным осям координат.
- Особая структура позволяет с большой скоростью находить ближайшие точки и кластеризовать пространство

### Структура 
- Чаще всего общая структура дерева/узлов состоит из:
  1. Информация о точке
  2. Ось деления 
  3. Указатели на следующие точки с другой осью деления (левую и правую часть)
  4. Некоторые Дополнительные {области} (см. ниже)

#### Дополнительно
- Чаще всего к основной Структуре дерева Добавляют дополнительные части, такие как _указатель на Родителя_ данная часть помогает ускорить скорость удаления и поиска ближайшей точки и соответственно кластеризовать.
- В большенстве случаев деление пространство происходит по следующему алгоритму: Сортировка по одной из осей $\rightarrow$ Поиск Медианы $\rightarrow$ Удаления из общего массива точек $\rightarrow$ Смена оси деления $\rightarrow$ Повторить  
  - Из за подобного способа записи, Дерево становиться сбалансированным 

### Функции 
1. Добавление
   - Добавление точки к существующему (или нет) дереву сохраняя деление пространства.
     - Добавление может происходить двумя способами:
       1. Перестройка всего дерева (более надёжно и стабильно, но **ОЧЕНЬ** долго)
       2. обход дерева и перестройка части дерева (Быстро, сложно логически в некоторых ситуациях не стабильно так как может перестать быть сбалансированным)
2. Удаление Узла
   - Удаление точки имеет те же два способа как в добавлении, однако при удалении можно сделать "_сдвиг_" и структура не нарушится 
3. Поиск 
   - Поиск происходит так же как и в бинарном дереве
4. Поиск ближайшего
    - Обход в ширину
5. Кластеризация
   - Есть два способа кластеризации, **dbScan** и **cMeans**
     1. dbScan принемает некий $\epsilon$ (растояние) и все точки в его пределах добавлает в общий кластер
     2. cMeans
